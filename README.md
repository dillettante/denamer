# denamer

**한국어 PDF 비실명화 도구 — 검은 박스로 "덮는" 것이 아니라 원문을 "삭제"합니다.**

공문서·계약서·신청서·보고서 같은 PDF에서 이름·주민등록번호·전화·계좌·카드·여권·주소를
탐지해 제거합니다. 모든 처리는 로컬에서만 이루어지며 네트워크 전송이 없습니다.

## 왜 "진짜 삭제"인가

PDF 위에 검은 사각형을 얹는 방식은 눈에만 안 보일 뿐, 원본 텍스트가 문서 안에
그대로 남습니다 — 복사·전체선택·OCR·LLM 투입 시 전부 노출됩니다. denamer는
PyMuPDF의 `apply_redactions()`로 해당 영역의 **텍스트 객체와 스캔 이미지 픽셀
자체를 삭제**한 뒤 그 자리에 표시(검은 박스 또는 대체어)를 그립니다. 저장된
PDF에서 원문 복원은 불가능합니다.

## 두 가지 모드

| 모드 | 이름 | 주소 | 번호류(주민·계좌·전화·카드) | 출력 파일명 |
|---|---|---|---|---|
| **익명화** `--mode anon` (기본) | 홍길동 → **홍OO** (성 보존) | 첫 토큰(시·도) 보존, 나머지 O | 검은 박스 완전 삭제 | `원본명_masked.pdf` |
| **가명화** `--mode pseudo` | 홍길동 → **C** (A·B·C… 일관 치환) | 익명화와 동일 | 검은 박스 완전 삭제 | `원본명_aliased.pdf` |

가명화는 **가명 대장**(`pseudonym_ledger.json`)으로 문서 간 일관성을 유지합니다 —
같은 대장을 쓰는 한, 여러 문서에서 같은 사람은 항상 같은 가명을 받습니다.
대장에는 실명↔가명 매핑이 들어가므로 **산출물과 함께 배포하면 가명화가 무효**입니다.
반드시 로컬에만 보관하세요.

## 설치·사용 (데스크톱판)

```bash
git clone https://github.com/dillettante/denamer.git && cd denamer
python3 -m venv --system-site-packages venv
./venv/bin/pip install pymupdf \
  "ko-pii @ git+https://github.com/Marker-Inc-Korea/ko-pii.git@635ade22cfe8d89761ed0e8948b5470e2307506e"
# 스캔 PDF 자동 OCR을 쓰려면: brew install ocrmypdf  (tesseract kor 포함)

./venv/bin/python b_redact.py 문서.pdf                # 익명화 → 문서_masked.pdf
./venv/bin/python b_redact.py 문서.pdf --mode pseudo  # 가명화 → 문서_aliased.pdf
./venv/bin/python test_b_redact.py                    # 회귀 테스트
```

- 원본 파일은 절대 수정하지 않습니다.
- 텍스트 레이어 없는 순수 스캔 PDF는 `ocrmypdf`(kor+eng)를 자동 실행한 뒤 처리합니다.
- `ko-pii` 미설치 시 정규식 탐지만으로 동작합니다(무라벨 자유문장 속 이름 탐지가 빠짐).
- 종료 코드: 잔존·매핑실패가 있으면 **1** — 이때 출력물을 신뢰하지 마세요.

## 웹판 (`denamer.html`)

단일 HTML 파일 —
[**여기서 내려받아**](https://raw.githubusercontent.com/dillettante/denamer/main/denamer.html)
브라우저에서 열고 PDF를 끌어놓으면 끝. 설치 불요, 오프라인 동작,
파일은 어디로도 전송되지 않습니다. 페이지를 이미지로 재조립하므로 출력에 텍스트
레이어 자체가 없습니다(검색 불가, 대신 복원도 불가능). 정규식 탐지만 사용하며
스캔 PDF(OCR 필요)는 데스크톱판을 쓰세요.

## 탐지 방식

1. **정규식 엔진** — 주민/외국인등록번호(자간·유니코드 대시·점 구분 변형 허용,
   생년월일 타당성 검증), 카드, 여권, 계좌(라벨-은행명-개행 관용), 전화(구분자
   필수 — 날짜 오인 방지), 이메일(줄바꿈 관용), 역할 라벨 문맥 이름(성명·담당자·대표자 등, 자간 허용), 라벨·지명 앵커 주소, 괄호 상세주소.
2. **ko-pii** (형태소 기반, 데스크톱판) — 라벨 없는 자유문장 속 이름·주소.
   오탐 억제: 성씨+이름끝음절+신뢰도 하한, 4자는 복성만, 고빈도 일반명사는
   강한 문맥 근거(직함·확정 PII 인접) 필수, 컨트롤 문자·개행 포함 값 거부.
3. **제거** — 탐지된 값 문자열을 페이지 전수 검색(리터럴 + 자간 변형 대응
   워드 폴백)해 모든 출현을 삭제. 오프셋→좌표 역매핑을 쓰지 않으므로 그 계열의
   좌표 어긋남 버그가 구조적으로 불가능.
4. **사후검증** — 저장본을 다시 열어 재검색. 잔존 발견 시 시끄럽게 실패(exit 1).

## 한계 — 반드시 읽으세요

- **자동 탐지는 완전하지 않습니다.** 룰·사전 기반이므로 서식·인코딩·OCR 품질에
  따라 놓치는 값이 있을 수 있고, 산문 문서에선 일반명사가 이름으로 오탐될 수
  있습니다(과잉 제거 방향 — 리포트의 `persons` 목록으로 확인).
- 사후검증은 "탐지한 값"만 재확인합니다. 탐지 자체가 놓친 값은 검증도 모릅니다.
- **외부 제출 전, 결과물을 원본과 대조하는 육안 검토가 이 도구 사용의 전제입니다.**
